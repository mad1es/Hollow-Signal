# Hollow Signal - MVP для iOS

## Описание

Hollow Signal — это интерактивный хоррор-опыт для iOS, который создаёт ощущение присутствия чего-то, что наблюдает за игроком через датчики телефона.

## Особенности

- **Распознавание речи** — игра реагирует на слова игрока
- **Отслеживание движений** — гироскоп и акселерометр фиксируют движения телефона
- **Анализ дыхания** — микрофон анализирует ритм дыхания
- **Датчик приближения** — реакция на расстояние до лица
- **Динамические реакции** — игра адаптируется к действиям игрока
- **Минималистичный интерфейс** — белый фон, чёрный текст, ничего лишнего

## Технические требования

- iOS 17.0+
- Xcode 15.0+
- Swift 5.0+
- Разрешения:
  - Микрофон
  - Камера (опционально)
  - Распознавание речи

## Структура проекта

```
HollowSignal/
├── HollowSignalApp.swift      # Точка входа приложения
├── ContentView.swift          # Главный экран
├── GameStateManager.swift     # Управление состоянием игры и фазами
├── SensorManager.swift        # Работа с датчиками (гироскоп, микрофон, приближение)
├── SpeechRecognizer.swift     # Распознавание речи
├── ReactionEngine.swift       # Система реакций на действия игрока
└── TextDisplayView.swift      # Компонент отображения текста
```

## Как запустить

1. Откройте проект в Xcode
2. Выберите симулятор или устройство iOS
3. Нажмите Run (⌘R)
4. При первом запуске разрешите доступ к микрофону и распознаванию речи

## Геймплей

Игра проходит через несколько фаз:

1. **Загрузка (0:00-0:05)** — чёрный экран, текст "подождите..."
2. **Первое взаимодействие (0:05-0:30)** — "Вы здесь?"
3. **Установление контакта (0:30-1:30)** — "Я пытаюсь понять"
4. **Наблюдения (1:30-3:00)** — игра комментирует состояние игрока
5. **Первое задание (3:00-5:00)** — "Скажите мне, что вы видите вокруг"
6. **Углубление контакта (5:00-7:00)** — "Постойте ближе"
7. **Аномалии (7:00-10:00)** — эффекты эха, предсказания движений
8. **Завершение (10:00+)** — финальные фразы

## ЛОР

Игра основана на концепции "сенсорной десинхронизации" — реального феномена, когда мозг неправильно обрабатывает сенсорную информацию. Подробнее см. `LORE.md`.

## Известные ограничения MVP

- Эффект эха голоса пока не реализован полностью (только текстовая реакция)
- Фейс-трекинг не реализован (требует Vision framework)
- История сообщений не сохраняется между сессиями
- Нет системы сохранений
